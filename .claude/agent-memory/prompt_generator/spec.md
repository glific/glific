# AI-Assisted Prompt Generator — Spec (v1, Beta)

## 1. Overview

Glific's Assistant creation page lets NGOs build AI-powered chatbots for WhatsApp. A key step in this flow is writing a system prompt (Instructions) — a skill many NGO users lack, often causing them to abandon setup before completing it.

This feature introduces an AI-assisted prompt generator as a low-fidelity beta addition to the existing Create Assistant page. By asking users 9 guided questions, the tool generates a ready-to-use, editable system prompt — removing the blank-page problem and helping NGOs get their first assistant live quickly.

## 2. Problem Statement

- NGO staff are not prompt engineers — writing a system prompt is technically unfamiliar and intimidating.
- Users get stuck at the Instructions field and either leave incomplete prompts or abandon the flow entirely.
- This creates friction right at the moment of highest excitement, reducing activation rates.
- There is currently no guidance, example, or scaffolding offered inside the product.

## 3. Proposed Solution

Add a "Generate with AI" button (labelled BETA) next to the Instructions (Prompt) field. Clicking it opens a step-by-step modal with 9 simple questions. On submission, a structured prompt is generated, shown in an editable preview, and applied to the Instructions field with one click.

*This is intentionally low-fidelity beta — the goal is to validate the concept, collect real usage data, and refine before building a more polished version.*

## 4. User Flow

| # | Step | Description |
| :---- | :---- | :---- |
| 1 | Trigger | User clicks "Generate with AI (BETA)" button next to the Instructions field on the Create Assistant page. |
| 2 | Question form | A modal opens with 9 guided questions covering name, purpose, audience, language, tone, format, restrictions, fallback, and escalation. |
| 3 | Generate | User fills answers (partial or full) and clicks "Generate Prompt". A 2–3 second loading state plays. |
| 4 | Review | A generated prompt appears in an editable text area with a notice: "This is an AI generated prompt. Please review and edit if required." |
| 5 | Apply | User clicks "Use this Prompt" — the prompt is inserted into the Instructions field and the modal closes. |
| 6 | Save | User reviews the full assistant form and saves as normal. |

## 5. The 9 Questions

These questions collect the minimum viable context needed to generate a high-quality system prompt. Each maps to a standard prompt-engineering decision.

| # | Question | Purpose |
| :---- | :---- | :---- |
| 1 | Name | Organisation name and desired chatbot name |
| 2 | Core purpose | One or two sentences describing what the assistant helps with |
| 3 | Audience | Who chats with the bot — literacy level, language familiarity, context |
| 4 | Language | Always English / Hindi / match user's language / regional / Hinglish |
| 5 | Tone | Simple & friendly / formal / instructional / other |
| 6 | Response format | Length (max lines/sentences) and structure (bullets, numbered, free text) |
| 7 | Off-limits | Topics the bot must never address (or NA) |
| 8 | Fallback | Exact text to show when the bot doesn't know an answer |
| 9 | Escalation | Phone, email, or next step for emergencies or out-of-scope queries (or NA) |

## 6. Scope — What's In & Out

| In Scope (Beta) | Out of Scope (Future) |
| :---- | :---- |
| Generate with AI button (BETA badge) on the Instructions field | Saving or versioning generated prompts |
| 9-question modal form with examples and progress bar | Prompt templates library or pre-fills by use-case |
| Loading state (2–3 sec) simulating AI generation | Prompt quality scoring or live suggestions |
| Editable generated prompt witha review notice | Auto-fill of other form fields from answers |
| "Use this Prompt" applies text to Instructions field | Multi-language UI for the question form |
| Confirmation notice after prompt is applied | Prompt regeneration within the modal after 1 prompt is shared |
| Usage metrics tracking (see Section 8) | Advanced persona or domain-specific tuning |

## 7. Design Decisions

- **BETA badge:** Signals to users this is experimental — manages expectations and reduces support load.
- **Editable prompt:** Users must feel ownership of the output — enforces review before applying.
- **Review notice:** "This is an AI generated prompt. Please review and edit if required" — builds trust and accountability.
- **Partial answers allowed:** No required fields on the question form — reduces friction and lets users try with minimal input.

## 8. Success Metrics

The following metrics should be instrumented from day one of the beta. They will form the evidence base for deciding whether and how to build the full feature.

| Metric | What we measure | Cadence | What it tells us |
| :---- | :---- | :---- | :---- |
| Button clicks | Count of clicks on "Generate with AI" button | Weekly | Is the feature being discovered? |
| Modal completion rate | % of users who answered all 9 questions before generating | Weekly | Are users engaging fully or dropping off? |
| Generation rate | % of users who click "Generate Prompt" after answering questions | Weekly | Are users trusting the flow? |
| Prompt applied rate | % of generated prompts where "Use this Prompt" is clicked | Weekly | Is the output useful enough to accept? |
| Prompt edit rate | % of applied prompts edited by the user before saving the assistant | Weekly | Is the generated prompt accurate / good enough? |
| Assistant save rate | % of sessions where the assistant is saved after using the generator | Weekly | End-to-end conversion — did the feature result in a live assistant? |
| Question drop-off | Which of the 9 questions are most often left blank | Monthly | Which questions cause friction or confusion? |
| Time to complete | Avg time from modal open to "Use this Prompt" click | Monthly | Is the flow fast and efficient? |
| Return usage | Number of users who use the feature more than once | Monthly | Is the feature valuable enough to reuse? |

## 9. Feedback Collection

Alongside quantitative metrics, qualitative feedback should be gathered to understand the NGO experience:

- Add an optional 1-question survey after "Use this Prompt" is clicked: "How useful was the generated prompt?" (1–5 stars)
- Conduct 3–5 user interviews with NGO staff within the first 4 weeks of launch
- Track support queries or Slack messages mentioning the prompt generator or Instructions field
- Note which of the 9 questions receive the most varied or unusual answers — these may need better examples or rewording
